## 用户故事 (User Stories)

| ID   | 用户故事 (Role + Want + Benefit)  | 验收标准 (Acceptance Criteria)    | 优先级      |
|:-----|:------------------------------|:------------------------------|:---------|
| US01 | 作为学生，我希望通过校园卡认证，以便确保交易环境安全。   | 支持上传照片或 OAuth 登录；后台人工/OCR 审核。 | **Must** |
| US02 | 作为发布者，我希望设置任务截止时间，以便过期任务自动下架。 | 超过截止时间未接单的任务状态自动变为“已失效”。      | **Must** |
| US03 | 作为接单者，我希望按宿舍楼栋筛选快递单，以便顺路代取。   | 筛选器包含：北区、南区、各宿舍楼选项且过滤准确。      | **Must** |
| US04 | 作为接单者，我希望在接单前看到发布者的信誉评分，避免被坑。 | 任务详情页显著位置显示发布者的历史星级评分。        | **Must** |
| US05 | 作为发布者，我希望即时收到接单提醒，以便进行沟通。     | 系统发送 App 内通知，点击通知直达聊天窗口。      | **Must** |
| US06 | 作为用户，我希望有一个内置聊天工具，以便保护隐私。     | 支持文字图片，订单结束后自动关闭/存档频道。        | Should   |
| US07 | 作为接单者，我希望能在地图上看到任务分布，以便规划路线。  | 提供地图视图，用气泡标注待接任务坐标。           | Should   |
| US08 | 作为管理员，我希望能够批量下架违规广告，维护平台环境。   | 管理后台支持勾选多个任务并点击“一键删除”。        | Should   |
| US09 | 作为用户，我希望支持微信/支付宝充值，以便灵活支付。    | 对接主流支付 SDK，充值 1 分钟内余额到账。      | Should   |
| US10 | 作为发布者，我希望支持“加急”功能，以便更快吸引接单。   | 支付额外费用后，该任务在列表中置顶或高亮。         | Could    |
| US11 | 作为接单者，我希望有“自动接单”助手，以便抢到特定单。   | 允许设置关键字，自动匹配并弹出强提醒。           | Could    |
| US12 | 作为用户，我希望有个人成就勋章，以便展示互助等级。     | 根据单数解锁“热心校友”、“互助达人”等称号。       | Could    |
| US13 | 作为发布者，我希望能指定特定群体可见（如仅限研究生）。   | 发布时有勾选框限制接单人的身份标签过滤。          | Could    |
| US14 | 作为管理员，我希望支持导出月度互助报告。          | 可生成 PDF 报告，包含交易总额、活跃人数等。      | Won't    |
| US15 | 作为用户，我希望支持语音输入需求，方便在路上操作。     | 集成语音转文字接口，自动填入描述栏。            | Won't    |

### AI 辅助实验回答
- **AI 是否遗漏了异常流？**
    - **回答**：是。AI 提供的初版用例仅包含“Happy Path”，遗漏了诸如“余额支付失败后的事务回滚”、“校园网高并发下的抢单冲突”以及“用户恶意评价”的异常流处理。
- **验收标准是否具体可测试？**
    - **回答**：否。AI 原始输出多为“系统应保证安全”等虚泛词汇。人工修改后的标准细化为“上传校园卡 2 小时内完成审核”、“点击通知 3 秒内跳转聊天”等可量化指标。
